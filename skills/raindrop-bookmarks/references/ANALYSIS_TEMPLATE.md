# Bookmark Analysis Template

Use this structure for all bookmark analysis files.

## Filename Format

```
x_bookmark_analysis_YYYY-MM-DD_{topic}_{author}.md
```

Example: `x_bookmark_analysis_2026-02-21_kelly_monte_carlo_mikita.md`

## Template

```markdown
# X Bookmark Analysis: {Title}

**Source**: {Platform} via Raindrop — {Author} ({handle})  
**Bookmarked**: {Date}  
**URL**: {URL}

---

## Summary

**Core Thesis**: {One sentence capturing the main idea}

**Context**: {Background on author/topic if relevant}

---

## Key Insights

### 1. {First Key Point}
{Explanation with specifics}

### 2. {Second Key Point}
{Explanation with specifics}

### 3. {Third Key Point}
{Explanation with specifics}

---

## Strategic Relevance

**For {Domain}**:
- {How this applies}
- {Actionable implications}

**For {Related Context}**:
- {Additional angles}

---

## Action Items

- [ ] {Specific action}
- [ ] {Specific action}
- [ ] {Specific action}

---

## Related Context

- {Author background}
- {Related bookmarks or resources}
- {Market/industry context}

---

**Tags**: analyzed, ghost-ai, {tag1}, {tag2}

*Analysis complete — imported via Raindrop sync*
```

## Tagging Convention

Always include:
- `analyzed` — marks as processed
- `ghost-ai` — identifies Ghost's analysis
- Topic tags: `trading`, `ai`, `hardware`, `coding`, etc.
